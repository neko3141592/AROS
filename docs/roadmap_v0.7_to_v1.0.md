# ARL 開発ロードマップ v0.7 〜 v1.0: Next.js UI と Electron Desktop でのリリース計画

## 目的
v0.6で構築したローカル FastAPI 基盤の上に、Next.js でUIを実装し、同一UI資産を活用して Electron デスクトップアプリへ展開する。最終的に v1.0 で安定リリース（GA）を達成する。

## 前提方針
- Backend は v0.6 の FastAPI + Worker + SQLite を継続利用
- FastAPI はクラウド公開せず、デスクトップアプリ内でローカル起動する
- UIは **Next.js を唯一のフロント実装** とし、Web版とDesktop版で共有
- Desktop版は Electron でラップし、配布・自動更新・ローカル体験を提供
- ログイン/マルチユーザー認証は導入せず、単一ユーザー利用を前提にする
- 重要な業務フロー（ジョブ投入、進捗監視、ログ閲覧、生成物確認、HITL操作）は Web/Desktop で同一体験に揃える

---

## v0.7: Next.js Web UI（MVP）

### 目標
ブラウザ上で、AROSジョブを「作成・監視・操作」できる最小UIを完成させる。

### 実装チェックリスト

#### 0.7.1 アプリ基盤
- [ ] `frontend/web` (Next.js App Router) を作成
- [ ] APIクライアント層（REST + WebSocket）を `lib/api` に集約
- [ ] 環境変数設計（`NEXT_PUBLIC_API_BASE_URL` など）
- [ ] FastAPIローカル起動確認（起動待ち・ヘルスチェック・再接続）

#### 0.7.2 ジョブUI
- [ ] ジョブ作成画面（Task Title / Description / Constraints）
- [ ] ジョブ一覧画面（状態、作成時刻、最終更新）
- [ ] ジョブ詳細画面（現在ノード、リトライ回数、結果）
- [ ] ログストリーム表示（WebSocket）

#### 0.7.3 生成物・HITL
- [ ] 生成コード閲覧（シンタックスハイライト）
- [ ] Research Context 表示
- [ ] HITLアクション（承認/停止/再実行）UI
- [ ] エラー時の再試行導線（ユーザーにわかる文言）

#### 0.7.4 品質
- [ ] 主要画面のE2Eテスト（Playwright想定）
- [ ] APIモックを用いたUI単体テスト
- [ ] 画面ロード性能の基準値設定（初回表示、ログ表示遅延）

### Exit Criteria (v0.7完了条件)
- [ ] ブラウザだけで1ジョブのライフサイクル（作成→実行→完了/失敗確認）を完遂できる
- [ ] 運用に必要な最低限の監視情報がUIから取得できる

## v0.8: Electron Desktop App（Beta）

### 目標
Next.js UIをElectronでデスクトップ化し、Web版と同等機能を提供する。

### 実装チェックリスト

#### 0.8.1 Desktopシェル
- [ ] `frontend/desktop` に Electron メインプロセスを作成
- [ ] 開発モード（Next.js dev server読み込み）と本番モード（ビルド成果物読み込み）の分離
- [ ] プラットフォーム別ビルド設定（macOS / Windows）

#### 0.8.2 セキュリティ・権限
- [ ] `contextIsolation: true`, `nodeIntegration: false` を徹底
- [ ] preload 経由の最小IPC設計（必要最小限）
- [ ] 設定情報の安全保存（ローカル設定ファイルの暗号化/権限制御）

#### 0.8.3 Desktop向け体験
- [ ] メニューバー/ショートカット（ジョブ再実行、ログコピー）
- [ ] ネイティブ通知（ジョブ完了/失敗）
- [ ] ログ/設定/キャッシュのローカル保存ポリシー

#### 0.8.4 配布準備
- [ ] インストーラー作成（electron-builder等）
- [ ] 署名（code signing）・配布チャネルの整備
- [ ] 自動更新（staged rollout を想定）

### Exit Criteria (v0.8完了条件)
- [ ] Web版と同じ主要機能をDesktop版で利用できる
- [ ] 開発者以外のマシンにインストールして動作確認できる
